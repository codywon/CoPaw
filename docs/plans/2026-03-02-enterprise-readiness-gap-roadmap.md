# CoPaw 企业级就绪差距路线图

日期：2026-03-02
负责人：平台/核心团队
状态：提案

## 1. 基线现状

当前状态是“企业可用 MVP”，但尚未达到“完整企业级”：
- 已具备多代理编排、角色策略、任务事件与基础审计能力。
- 默认安全模型仍是单操作者信任边界。
- 仍使用进程内队列与文件任务存储。
- 完整 ABAC/SSO、实时可观测性栈、租户级治理能力尚未完善。

## 2. 企业级目标定义

若要宣称“企业级”，平台应满足：
1. 身份与访问控制：至少 SSO + RBAC，并可扩展到 ABAC。
2. 租户隔离：跨租户数据面与控制面的严格隔离。
3. 审计与合规：防篡改审计链路 + 保留策略 + 导出能力。
4. 可靠性：HA 部署、备份恢复、明确 RPO/RTO。
5. 可观测性：指标/日志/链路追踪 + 基于 SLO 的告警。
6. 治理能力：按租户对限流/配额/预算进行策略治理。

## 3. 交付路线

## 阶段 A（P0）：身份认证与访问控制基线（2-4 周）

### 范围
- 接入 SSO（优先 OIDC，后续可选 SAML）。
- 建立 RBAC 模型：
  - 角色：`platform_admin`、`tenant_admin`、`operator`、`viewer`
  - 覆盖配置、渠道、技能、MCP、定时任务、任务取消、模型策略变更等权限。
- 在 Console API 和关键后端路由统一执行鉴权。
- 加固会话与令牌策略（过期、轮转、安全默认值）。

### 关键交付物
- 鉴权服务模块与中间件。
- 路由权限守卫（装饰器/拦截器）。
- RBAC 权限矩阵文档。
- 关键敏感接口 E2E 权限测试。

### 验收标准
1. Console/API 对未认证访问受保护路由返回拒绝。
2. 同一接口在权限不足时返回 `403`，权限充足时返回 `200`。
3. 角色授权变更无需重启即可生效。
4. 安全回归测试在 CI 中通过。

## 阶段 B（P0）：租户隔离与治理（3-5 周）

### 范围
- 将租户升级为一等领域对象。
- 完成租户资源隔离：
  - 配置、凭证、任务历史、记忆、技能/MCP 绑定、定时任务。
- 在任务派发中引入租户上下文绑定。
- 增加租户级策略：
  - 每租户限流
  - 每租户配额（日/月）
  - 每租户模型预算护栏。

### 关键交付物
- 请求/会话/Runner 全链路租户上下文透传。
- 存储模型升级（关键记录引入 tenant_id）。
- 策略引擎扩展（租户范围校验）。
- 租户配额与预算的管理 API/UI。

### 验收标准
1. 默认阻断跨租户读写。
2. 压测下配额与限流生效且行为正确。
3. 租户预算超限可观测、可处置（事件 + API 状态）。
4. 租户隔离测试通过（含负向用例）。

## 阶段 C（P1）：HA/容灾 + 可观测性 + 合规控制（4-6 周）

### 范围
- 替换本地运行瓶颈：
  - 队列：`InProcQueue -> RedisQueue`
  - 任务与审计存储：文件 -> PostgreSQL/对象存储方案。
- 增加 HA 部署形态：
  - 应用无状态多副本
  - 单实例任务的 leader/租约保护
  - 健康检查、就绪检查、启动检查。
- 建设可观测性栈：
  - 指标（Prometheus）
  - 日志（Loki/ELK 兼容）
  - 链路（OpenTelemetry/Jaeger 兼容）
  - 基于错误率、队列积压、超时比例的 SLO 告警。
- 增加合规导向控制：
  - 不可变审计导出链路
  - 保留与归档策略
  - 密钥管理与机密处理基线。

### 关键交付物
- 生产部署蓝图（单地域 HA 基线）。
- 备份/恢复 Runbook 与容灾演练流程。
- 核心链路告警规则与观测看板。
- 合规证据清单（运行控制 + 日志证据）。

### 验收标准
1. HA 故障切换测试通过，关键数据丢失不超过声明 RPO。
2. 恢复演练达到声明 RTO。
3. 核心 SLO 看板与告警上线且可执行。
4. 审计导出可查询，防篡改机制有文档化证明。

## 4. 优先级

- P0（企业上线前必须具备）：阶段 A + 阶段 B。
- P1（规模化与合规评审前必须具备）：阶段 C。
- P2（增强项）：ABAC 策略、高级审批流、跨地域容灾。

## 5. 建议里程碑

1. M1（第 2 周）：OIDC 登录 + RBAC 守卫在预发环境可用。
2. M2（第 5 周）：租户隔离与配额策略在预发环境可用。
3. M3（第 8 周）：Redis/PostgreSQL 后端替换与迁移脚本完成。
4. M4（第 10 周）：可观测性 + SLO 告警 + 容灾演练完成。
5. M5（第 12 周）：企业级就绪评审与上线清单签署完成。

## 6. 风险登记

1. 鉴权改造触达面广（大量 API）。
   - 缓解：按路由渐进接入守卫 + 兼容窗口。
2. 租户化迁移风险（数据模型变更大）。
   - 缓解：幂等迁移脚本 + 影子校验。
3. HA/容灾提升运维复杂度。
   - 缓解：标准化 Runbook + 自动化演练 + 值班手册。
4. 策略收紧可能影响现有自动化流程。
   - 缓解：分阶段策略模式（`report-only` -> `enforce`）。

## 7. 下一迭代建议（立即执行）

1. 冻结 RBAC 权限矩阵与受保护 API 清单。
2. 实现 OIDC 鉴权中间件与会话/令牌基线。
3. 在 `config`、`providers`、`mcp`、`subagents`、`cron` 路由接入角色校验。
4. 在 CI 增加鉴权回归与隔离负向测试骨架。
